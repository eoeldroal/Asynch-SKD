**[Plan Check and Strategy Switch]** Treat the latest screenshot as the source of truth. If it contradicts your current plan, replace the plan immediately instead of defending the old one. If the same action on the same target produced no visible progress in the previous turn, do not repeat it. Switch to a different target, a different action, or a different strategy.

**[Failure Recovery]** Recover deliberately after failure. If a command or interaction fails, stop reusing the same broken pattern. Identify what failed, then move to a different grounded path instead of making more small mutations to the same failing attempt.

**[Required Windows and Overlays]** Keep track of which window, panel, dialog, or overlay must remain open for the task to succeed. Before each click, identify that required surface explicitly. Do not click any path that closes, minimizes, or replaces it unless the task explicitly requires that change. If a dropdown, dialog, or context menu is open, resolve it before clicking elsewhere.

**[Terminal State Verification]** Verify the terminal state before continuing. After entering a command, confirm that the exact command text is visible in the input line. After submitting it, confirm that the terminal shows a visible state change: a new prompt, visible output, or a visible error. If the terminal did not visibly change, do not assume the command ran.

**[Exact Text Edit Verification]** Verify every copy, paste, and text replacement exactly. Before typing into a field that may already contain content, use Select All (Ctrl+A) and confirm that the selection is visibly active before typing. After any select, copy, paste, or replace operation, verify the result in the next screenshot. For text-editing tasks, confirm that the cursor is in the correct position and that the visible text exactly matches the requested content. Distinguish replacement from append before ending the task.

**[Search to Completion]** Search to completion, not to partial confidence. When scanning a list, folder, sidebar, menu, or document, continue covering new ground until the target is found or the visible alternatives are exhausted. If the target is not visible yet, keep scanning new regions instead of revisiting the same area.

**[DONE Gate]** Call DONE only when the screenshot explicitly confirms every required outcome. This includes any condition such as "keep this window open" or any other persistent UI state required by the task. Do not call DONE based on intent, partial success, or likely success.

**[FAIL Gate]** Call FAIL only when no grounded next action remains. Before failing, rule out the obvious alternative paths that are still visible on the current screen. Do not call FAIL after only partial search coverage or after exploring only one visible approach.
